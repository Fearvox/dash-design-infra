# @dash/scale

Fluid type and space scale generator. Wraps `utopia-core` with fixed-canvas editorial defaults:

- Canvas min/max tuned to our landscape A-series reverse-beat (1280 preview → 1684 print)
- φ (1.618) type ratio
- Baseline snap enforced (all line-heights must be multiples of 8px)

## Use

### Dry-run the scale

```bash
bun gen
```

Prints the generated type + space scale to stdout as JSON.

### Write back to @dash/tokens

```bash
bun gen --write
```

Writes the V4 tuned type scale into `@dash/tokens/src/tokens.json` and rebuilds `@dash/tokens`.
The merge is targeted: it only replaces `font.size.*.$value` fields and preserves the hand-edited token file shape, descriptions, and ordering.

### Programmatic use

```ts
import { snapLineHeight, generateTypeScale } from '@dash/scale';

snapLineHeight(18.6); // → 16
generateTypeScale();   // → utopia-core output
```

## Why utopia-core, not utopia.fyi web app

The web app is a generator that outputs CSS clamp() strings. Utopia-core is the same math as a pure TS package. We pin it so scale generation is:
- Reproducible (same inputs → same output, no browser state)
- Version-controlled (upgrades are intentional)
- Composable (can feed into tokens, capsize, measure)

## Design decisions

- **Landscape canvas is fixed, not fluid**. Font-sizes are generated by the same math but clamped to a single size — the fluid-clamp() output is kept as a future-proof option for when we add mobile or multi-canvas surfaces.
- **Baseline snapping is non-negotiable**. `snapLineHeight(px)` always returns a multiple of 8. Non-multiples are a bug.
- **Merge-back is targeted**. `--write` only updates `font.size` values. Space tokens remain unchanged because the current 4-80px ladder already matches the generated baseline scale.
