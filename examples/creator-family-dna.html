capsule grammar card

The DASH visual DNA is not a single palette — it is a contract of constraints. Every surface must choose a family and declare its tokens in :root. The grammar is: dark-industrial for monitoring, warm-paper for editorial, creative-hybrid for explainers. Drift is allowed for creative role; nameless hardcoded values are not.

1. Every surface has a :root

No hardcoded colors in selectors. Every color must pass through a CSS custom property declared in :root and referenced via var().

✗ touchdesigner-tox: zero CSS custom properties — 12 hardcoded values.

2. Choose a family, declare its tokens

Pick warm-paper, dark-industrial, or creative-hybrid. Declare the family's token set in :root. Converge to consensus unless creative divergence is intentional.

✓ --blue→#1d4ed8 in warm-paper; --blue→#8db4ff in creative-hybrid.

3. Naming is part of grammar

Same concept, same name. Don't use --bg in one surface and --night in another for the same role. Don't interchange --signal/--accent/--blue.

✗ 6 names for "background", 3 for "primary accent", 3 for "error".

4. Machine-checkable convergence

bun creator:surface-consistency audits every surface's :root. This card defines the consensus target it measures drift against.

✓ Consensus → warm-paper majority. Consensus pill: paper #f2eadb · ink #15120e · muted #6f6659 · blue #1d4ed8 · amber #c47a1f · green #2f6f4e · line rgba(21,18,14,0.18).
